# cPanel Redis Manager - Feature Reference

Complete feature list for current and upcoming releases.

---

## Current Release - v2.0.4

> Stable base release - available to early adopters and new installations.

### One-Click Redis Management

Redis instances are managed directly from the cPanel interface.

- Start, stop, and restart with a single click
- No SSH access required
- Automatic initialization on first run

---

### Automatic Port Assignment

Each account receives a dedicated, conflict-free port.

- No shared ports
- No manual configuration
- Stored and tracked per instance

---

### Secure AUTH Key Generation

Each instance uses a unique authentication key:

- Generated via `random_bytes()`
- Stored with restricted permissions (`0640`)
- Never embedded in source code

---

### Per-Account Instance Isolation

Each cPanel account runs its own Redis process:

- Runs under the account system user
- Bound to `127.0.0.1`
- Fully isolated config, data, and logs

No cross-account access is possible.

---

### Automatic Configuration Generation

Configuration is generated automatically on first start.

Defaults can come from WHM or built-in values.

**Default example:**
bind 127.0.0.1
maxmemory 1536mb
maxmemory-policy allkeys-lfu
databases 5
requirepass <auto-generated>

---

### Health Monitoring

Instance status is determined through active checks:

- TCP connectivity validation
- authenticated Redis response validation

This ensures accurate detection of crashed or unresponsive instances.

---

### Auto-Restart via Cron

Each account includes a monitoring cron job:

- Automatically restarts Redis if it crashes
- Does not restart intentionally stopped instances
- Controlled via state tracking

---

### Activity Logging

All actions are logged per account:

- start / stop events
- config generation
- cron updates

Logs are stored in:

~/.cpanel/plugin/redis/log/


---

### CloudLinux and CageFS Support

Fully compatible with:

- CloudLinux
- CageFS isolation
- Alt-PHP environments
- PHP Selector

---

### Multi-PHP Support

Works with:

- EasyApache 4
- CloudLinux alt-PHP

Redis PHP extension must be installed separately.

---

### License System

- License key stored securely (`0600`)
- Validation performed against license server
- Local caching avoids runtime dependency
- Grace period ensures availability during outages
