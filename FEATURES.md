# cPanel Redis Manager — Feature Reference

Complete feature list for the current release and upcoming versions.

---

## Current Release — v2.0.4

### One-Click Redis Management

Redis instances are started and stopped directly from the cPanel interface with a single button. No SSH access is required for end users. The first initialization sets up all configuration automatically.

### Automatic Port Assignment

Each account receives a dedicated, conflict-free port assigned at provisioning time. The port is recorded in the instance configuration and is not shared with other accounts.

### Secure AUTH Key Generation

A cryptographically random authentication key is generated per instance using `random_bytes()`. Keys are stored in the per-user configuration file with restricted permissions (`0640`). They are never embedded in plugin source files.

### Per-User Instance Isolation

Every cPanel account runs its own Redis process, bound to `127.0.0.1`. Accounts cannot access each other's instances. Data directories, config files, log files, and PID files are all scoped to the individual user's home directory.

### Automatic Configuration Generation

On first start, the plugin generates a complete Redis configuration file tailored to the account. Default values are applied from the server-wide WHM configuration if set by the administrator, otherwise sensible built-in defaults are used.

**Default configuration:**
```
bind            127.0.0.1
maxmemory       1536mb
maxmemory-policy allkeys-lfu
databases       5
requirepass     <auto-generated>
```

### Health Monitoring

Instance status is determined by an authenticated TCP port check rather than relying on PID files. This means status is always accurate — if the Redis process has crashed, the plugin detects it correctly.

### Auto-Restart via Cron

A per-user cron job monitors the instance and restarts it if it goes down unexpectedly. The cron job is state-aware — if an admin or user intentionally stops the instance, the cron will not restart it. A state file controls this behavior.

### Activity Logging

All actions — start, stop, config creation, cron changes — are written to a per-user log file at `~/.cpanel/plugin/redis/log/<username>.log`. Log files persist across upgrades.

### CloudLinux and CageFS Support

The plugin is fully compatible with CloudLinux environments including:
- Alt-PHP with Redis extension support
- PHP Selector integration
- CageFS isolation compatibility
- `cagefsctl` registration during installation

### Multi-PHP Support

Works with all PHP versions available on the server, including EasyApache 4 managed PHP and CloudLinux alt-PHP. PHP extension installation instructions are provided for both environments.

### UHREDIS-KEY Licensing System

Two key formats are supported:
- `UHREDIS-` — standard license, validated by key only
- `UHREDISIP-` — IP-locked license, validated against the server's public IP

License keys are stored in a protected `.license` file with `chmod 600`. They are never embedded in PHP source code.

## v2.1.0 — WHM Plugin (In Development)

### WHM Admin Dashboard

A full server-wide management interface in WHM showing all cPanel accounts with Redis instances, their status, resource usage, and available actions.

### Real-Time Performance Charts

Memory usage and connection counts per account displayed as live Chart.js bar charts. Data sourced directly from `redis-cli info all` — not simulated.

### Password Rotation

Administrators can rotate the authentication key for any account's Redis instance from WHM. The change is applied live via `CONFIG SET` without a restart, then written to the config file and root metadata store simultaneously.

### Web-Based Redis CLI

An in-browser Redis CLI for executing commands against any account's instance. Commands are executed via `proc_open()` with an explicit argument array — no shell string interpolation. An allowlist of safe Redis verbs is enforced at the server level.

### Global Configuration Editor

WHM administrators can set server-wide defaults for new Redis instances:
- Default `maxmemory`
- Default eviction policy
- Default number of databases
- CLI access toggle per account

### CSF Firewall Integration

Automatically registers Redis ports with CSF's ignore list to prevent firewall interference with localhost-bound ports.

### Global Backup System

Trigger `BGSAVE` on all running instances simultaneously or per account. Results reported per account with success/failure status.
