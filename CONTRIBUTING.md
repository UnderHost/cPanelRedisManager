# Contributing to cPanel Redis Manager

Thank you for your interest in contributing to the cPanel Redis Manager project.

---

## Scope of This Repository

This repository contains **documentation only**. The cPanel Redis Manager plugin is a proprietary commercial product developed and maintained by [UnderHost](https://underhost.com). The plugin source code is not publicly distributed, and code contributions are outside the scope of this repository.

---

## How You Can Contribute

### Documentation Improvements

If you find errors, outdated information, or unclear explanations in any documentation file, we welcome pull requests.

**To contribute a documentation fix:**
1. Fork this repository
2. Create a branch: `git checkout -b fix/description-of-change`
3. Make your changes
4. Commit with a clear message: `git commit -m "docs: fix typo in installation overview"`
5. Open a pull request with a brief explanation of what you changed and why

Documentation contributions should:
- Be accurate and consistent with the official product website at [cpanelredismanager.com](https://cpanelredismanager.com)
- Follow the existing tone and style (technical, clear, no filler)
- Not include download links, installation scripts, or references to private distribution endpoints
- Not include invented features, pricing, or capabilities not officially announced

### Bug Reports and Feature Requests

If you encounter a problem with the installed plugin or have a feature suggestion:

1. Check the [existing issues](https://github.com/UnderHost/cPanelRedisManager/issues) first
2. Open a new issue with as much context as possible
3. For plugin bugs, include your OS, cPanel version, and Redis version
4. For feature requests, describe the use case clearly

### Security Vulnerabilities

**Do not report security vulnerabilities through GitHub Issues.** See [SECURITY.md](SECURITY.md) for the responsible disclosure process.

---

## What Is Not In Scope

- Pull requests that add or modify plugin source code
- Pull requests that attempt to document private installation endpoints
- Issues that request the plugin to be made open source

---

## Code of Conduct

Be respectful and constructive. We are a small team and genuinely appreciate feedback that helps improve the product and its documentation.
